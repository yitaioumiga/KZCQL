## 30秒理解全局

KZCQL（卡兹克创作质量闭环）是公众号长文创作多智能体系统，核心流程：

```
写作前必问（5项） → khazix-writer 撰写 → 去AI感检测 → 后置评审 → 发布
```

**核心风格**：「有见识的普通人在认真聊一件打动他的事。」

**文件结构**：
```
KZCQL/01_共享知识库/
├── 前置撰写规则/
│   ├── 前置撰写规则.md      ← 最核心，写作前必读
│   └── 去AI感检测清单.md     ← 写完后自检
└── 后置评审规则/
    ├── 后置评审规范.md       ← 评审流程
    ├── 一票否决项.md         ← 评审第一步检查
    ├── 评分体系.md           ← 评审打分标准
    └── 快速执行手册.md       ← 你正在看的文件
```

## 三种执行场景

### 场景一：仅评审（已有文章，需要评审）

```
1. 读取《一票否决项.md》，逐项检查
   ↓ 命中任一项 → 退回，结束
2. 读取《去AI感检测清单.md》，执行 K1-K5 检测
3. 读取《评分体系.md》，按 12 个维度打分
4. 撰写评审报告（格式见《后置评审规范.md》）
5. 总分 >= 85 → 通过；< 85 → 附修改建议退回
```

### 场景二：评审 + 撰写（需要写新文章并评审）

```
1. 读取《前置撰写规则.md》§8 写作前必问清单
   ↓ 向用户确认 5 项必问
2. 读取《前置撰写规则.md》§9 用户身份档案
3. 调用 khazix-writer Skill 撰写文章
4. 执行去AI感检测（K1-K5）
5. 执行后置评审（一票否决 → 12维度评分）
6. 总分 >= 85 → 交付；< 85 → 修改后重新评审
```

### 场景三：完整流程（从零开始到发布）

```
1. 【写作前】
   - 读取《前置撰写规则.md》全文
   - 执行 §8 写作前必问清单（5项）
   - 确认 §9 用户身份档案
   - 判断文章原型（§2.4.7）

2. 【撰写中】
   - 调用 khazix-writer Skill
   - 遵守 §1-§7 所有规范
   - 生成配图（§4 文艺插图 + §5 架构图）
   - 同时输出 .md 和 .docx（§7）

3. 【自检】
   - 执行四层自检（§2.4.6 L1-L4）
   - 执行《去AI感检测清单.md》全部检测项

4. 【评审】
   - 执行《一票否决项.md》检查
   - 执行《评分体系.md》12维度评分
   - 撰写评审报告

5. 【交付】
   - 总分 >= 85 → 交付文件
   - 文件清单：.md + .docx + images/
```

## 评分速查卡

| 维度 | 满分 | 核心标准 |
|------|------|----------|
| D1 标题质量 | 8 | 不超过20字，有画面感/悬念感 |
| D2 开头吸引力 | 10 | 场景代入，前文衔接自然 |
| D3 内容深度与价值 | 15 | 观点有支撑，知识输出方式好，文化升维 |
| D4 结构完整性 | 10 | 有小标题，章节30-50行，逻辑连贯 |
| D5 语言风格与质感 | 15 | 去AI味，口语化，卡兹克风格一致 |
| D6 加粗金句 | 5 | 3-5句，章节末尾，15字以内 |
| D7 配图质量 | 10 | 文艺插图无文字，风格统一，位置正确 |
| D8 结尾收束力 | 7 | 场景代入式互动，不生硬，无署名 |
| D9 事实准确性 | 10 | 关键事实已核查，无过时信息 |
| D10 技术规范 | 10 | .md+.docx双格式，文件命名正确 |

**评级**：S(95-100) / A(85-94) / B(75-84) / C(65-74) / D(60-64) / F(<60)
**通过门槛**：>= 85 分（A级）

## 一票否决速查卡

| 编号 | 名称 | 一句话判定 |
|------|------|-----------|
| V-01 | 虚构事实 | 编造了不存在的事实/数据/引用 |
| V-02 | 事实冲突 | 文章内部自相矛盾 |
| V-03 | 证据断链 | 核心论点缺乏支撑证据 |
| V-04 | 开头失真 | 编造了不存在的开头场景 |
| V-05 | 元指令泄漏 | 暴露了 AI 写作痕迹 |
| V-06 | 结构伪造 | 结构完整但内容空洞 |
| V-07 | 抄袭洗稿 | 大段复制他人内容 |
| V-08 | 标题超20字 | 标题字数超过20字 |
| V-09 | AI味严重 | 禁用词>5个 或 标点违规>3处 |
| V-10 | 事实核查失败 | 关键事实错误（星数/版本号/论文） |
| V-11 | 文末署名/邮箱 | 结尾出现作者署名或投稿邮箱 |
| V-12 | 违反身份设定 | 场景不符合用户真实身份 |
| V-13 | 事实漂移 | 将不确定信息写为确定事实（预计写为确定、约数写为精确、可能写为一定） |
| V-14 | 焦虑煽动 | 制造对立、贩卖焦虑、使用恐吓性语言操纵读者情绪 |

**命中任一项 → 直接退回，不进入评分。**

## 迭代策略速查

| 修改幅度 | 评审等级 | 说明 |
|----------|----------|------|
| 首次提交 | R2（标准评审） | 全文 12 维度评分 |
| 修改 < 30% | R3（轻量评审） | 仅检查修改部分 |
| 修改 >= 30% | R2（标准评审） | 全文重新评分 |

**迭代上限**：同一篇文章最多迭代 5 轮（与主Agent编排指南一致）。5 轮后仍未达到 85 分，建议重写。

## 前置规则速查

### 标题
- 不超过 20 字
- 有画面感/悬念感/反差感

### 语言
- 禁用词 18 个（说了算、说白了、意味着什么...）
- 标点禁令：不用冒号、破折号、双引号
- 口语化词组至少 8-10 个
- 加粗金句 3-5 句，15 字以内

### 结构
- 长文章必须有章节标题（用加粗行内标题代替 ##）
- 章节 30-50 行
- 开头：场景代入 + 前文衔接
- 结尾：场景代入式互动，无署名无邮箱

### 配图
- 文艺插图：4-5 张，无文字，16:9，手绘风
- 架构图：按需 2-4 张，arch- 前缀，.png
- 结尾不配图
- 总计 6-9 张

### 输出
- 必须同时输出 .md 和 .docx
- 配图在 images/ 目录

## khazix-writer Skill 调用速查

### 调用时机
- 用户要求撰写公众号文章时自动触发
- 触发词：写文章、写稿子、帮我写、续写、扩写、公众号文章、长文

### 调用前准备
1. 读取《前置撰写规则.md》
2. 执行 §8 写作前必问清单
3. 确认 §9 用户身份档案
4. 判断文章原型（§2.4.7）

### Skill 内冲突解决
| 冲突点 | Skill 要求 | 创作规范要求 | 解决方案 |
|--------|-----------|-------------|----------|
| 小标题 | 不加小标题 | 长文章必须有 | 用加粗行内标题代替 ## |
| 结尾格式 | Skill 有固定格式 | 场景代入式结尾 | 用我们自己的结尾 |
| 标点 | 不用冒号/破折号/双引号 | 同 | 一致，遵守 |

### 写完后必须执行
1. L1 硬性规则检查（禁用词 + 禁用标点 + 结构套话 + 空泛工具名）
2. L2 风格一致性检查（开头 + 节奏 + 口语化 + 标点二次确认）
3. L3 内容质量检查（观点支撑 + 知识输出 + 文化升维 + 对立面）
4. L4 活人感终审（温度感 + 独特性 + 姿态 + 心流）
5. 《去AI感检测清单.md》全部检测项
