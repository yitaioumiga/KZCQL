# 后置评审规范

> KZCQL 系统后置评审规则文件。评审 Agent 在执行评审时必须完整读取本文件。
>
> **适用范围**：公众号长文评审

> **版本**：v1.1  
> **关联文件**：评分体系.md、一票否决项.md、快速执行手册.md、系列文章评审检查项.md

## 一、评审触发条件

### 1.1 差异评审触发

根据文章修改幅度选择评审等级：

| 修改幅度 | 评审等级 | 说明 |
|----------|----------|------|
| 修改 < 30% | **R3（轻量评审）** | 仅检查修改部分，关注是否引入新问题 |
| 修改 >= 30% | **R2（标准评审）** | 全文评审，使用完整评分体系 |

### 1.2 首次评审

新文章首次评审统一使用 **R2（标准评审）**，按完整评分体系逐维度打分。

## 三、「引入新问题」判定标准

在 R3（轻量评审）中，核心任务是检测修改是否引入了新问题。

### 3.1 什么算「引入新问题」

| 类别 | 具体表现 | 严重程度 |
|------|----------|----------|
| 事实错误 | 修改后引入了新的错误事实（版本号、星数、人名等） | 严重 |
| 事实精度偏差 | 修改后数字/时间与素材原文或创作者确认不一致 | 严重 |
| 时间锚点错误 | 相对时间或绝对时间与发文日期不匹配 | 严重 |
| 受众措辞滞后 | 受众描述与最新定位报告不一致 | 中等 |
| 平台细节错误 | 平台名称或平台特性与实际不符 | 中等 |
| 禁用词回潮 | 修改过程中重新引入了已删除的禁用词 | 中等 |
| 标点违规 | 修改后出现了冒号、破折号、双引号 | 中等 |
| 风格不一致 | 修改部分的风格与原文不统一 | 轻微 |
| 结构破坏 | 修改导致章节长度失衡或逻辑断裂 | 中等 |
| 配图失效 | 修改后配图与文字内容不对应 | 轻微 |
| 新增AI味 | 修改部分出现了新的AI味表达 | 中等 |

### 3.2 判定流程

```
1. 对比修改前后的差异
   ↓
2. 检查差异部分是否命中「引入新问题」的任一类别
   ↓
3. 如命中「严重」类别 → 退回修改，不计入评审通过
   ↓
4. 如命中「中等」类别 → 标注问题，要求修改后重新提交
   ↓
5. 如命中「轻微」类别 → 标注建议，可计入评审通过
   ↓
6. 无命中 → R3 评审通过
```

## 五、评审执行流程

```
1. 读取文章全文
   ↓
2. 读取前置撰写规则，确认规范要求
   ↓
3. 执行一票否决检查（读取《一票否决项.md》）
   ↓ 如命中 → 直接退回，不进入评分
4. 执行去AI感检测（读取《去AI感检测清单.md》）
   ↓
5. 按维度逐项评分（读取《评分体系.md》）
   ↓
6. 汇总评分，计算总分和评级
   ↓
7. 撰写评审报告
   ↓
8. 如总分 >= 85 → 通过
   如总分 < 85 → 不通过，附修改建议
```
