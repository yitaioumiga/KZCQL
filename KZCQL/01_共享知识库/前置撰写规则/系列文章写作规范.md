# 系列文章写作规范

> 本规范适用于所有系列文章的撰写。W1 在撰写系列文章时必须读取本文件。

---

## 一、核心原则

### 1.1 独立可读

每篇文章必须**独立成篇**，不能假设读者看过系列其他文章。

- 禁止：「上期我们讲了xxx，这里就不重复了」
- 禁止：「关注我们这个系列的朋友应该知道」
- 正确：对前文提到过的概念，用1-2句话简要说明，让新读者也能跟上

### 1.2 系列连贯

虽然每篇独立，但系列整体要有**连贯性**。

- 每篇开头可以自然引用前文（「还记得那个xxx的类比吗」「之前聊过xxx」）
- 但不要大段复述前文内容
- 引用前文时，要给出足够上下文，让没看过的读者也能理解

### 1.3 身份一致

系列所有文章中的**用户身份设定**必须一致。

- 在系列计划文件中明确身份设定（学生/职业/工具使用经验等）
- 每次写作前读取确认
- 身份描述矛盾是严重问题

---

## 二、结构规范

### 2.0 系列标识（必须）

**每篇文章末尾必须有系列标识**，放在下期预告之后、自检报告之前。

**格式**：

```
> *系列名称 · 第X篇/共Y篇*
```

**规则**：
- 标识放在下期预告段落之后，用引用块+斜体呈现，视觉上与正文区分
- 格式为「系列名称 · 第X篇/共Y篇」
- 系列名称使用系列计划文件中的正式名称
- **禁止在文章开头或标题中标注篇数**，避免影响新读者完读率和平台推荐

**为什么放文末不放开头**：
- 开头标注篇数会让新读者产生「没看前文就看不懂」的心理门槛，降低完读率
- 公众号推荐算法偏好独立文章，开头标注系列信息可能被降权
- 文末标注既保持开头干净，又给老读者仪式感和追更锚点

### 2.1 下期预告（必须）

**每篇文章末尾必须有下期预告**，包含以下要素：

| 要素 | 说明 | 示例 |
|------|------|------|
| 下一期主题 | 一句话概括下期内容 | 「下期聊聊八维度架构审查框架」 |
| 悬念/钩子 | 让读者有期待感 | 「为什么需要独立的督察智能体？」 |
| 新系列预告（如有） | 正在筹备的新系列 | 「另外，理财访谈系列即将上线」 |

**格式**：在金句清单之后，用分隔线隔开，标题为「下期预告」。

### 2.2 往期链接（视情况）

当文章**依赖前文概念**时，必须在文中给出往期文章链接。

| 场景 | 处理方式 |
|------|----------|
| 引用前文核心概念 | 1-2句话简要说明 + 往期文章标题 |
| 深度依赖前文结论 | 简要说明 + 「详见《xxx》」 |
| 仅作为背景提及 | 不需要链接 |

**注意**：链接以文章标题形式给出，不使用URL（公众号文章链接在发布时生成）。

### 2.3 收官总指引（最后一篇）

**系列最后一篇文章**必须包含系列总指引，放在文章末尾（预告之后）。

格式：

```
**系列导航**

第1篇：《xxx》—— 一句话概括
第2篇：《xxx》—— 一句话概括
...
第N篇：《xxx》—— 一句话概括（本篇）
```

### 2.4 承接上期（非首篇）

**非首篇文章**开头应自然承接上期，但不要用模板化句式。

- ❌ 模板化：「在上期文章中，我们探讨了...」
- ✅ 自然：「还记得那个74分的审查报告吗？」「上次聊到R3差异评审...」

承接的目的是让老读者感到连续性，同时不让新读者困惑。

## 三、系列计划文件

### 3.1 文件位置

每个系列在 `04_工作区/产出归档/` 下有独立的日期文件夹，文件夹内应包含系列计划文件。

### 3.2 计划文件内容

| 字段 | 说明 |
|------|------|
| 系列名称 | 系列的正式名称 |
| 系列定位 | 目标读者、核心主题、文章风格 |
| 文章清单 | 每篇的标题/主题/状态（计划中/撰写中/已发布） |
| 用户身份设定 | 系列中用户的身份描述 |
| 关键概念索引 | 系列涉及的核心概念及其首次出现的文章 |

### 3.3 同步更新

每完成一期文章，必须同步更新系列计划文件中的状态和文件路径。

- [ ] 涉及前文概念时有简要说明
