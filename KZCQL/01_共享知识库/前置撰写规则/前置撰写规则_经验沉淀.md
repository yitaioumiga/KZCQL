## §19 人工校验四原则（定位宣言v5微调经验沉淀）

> 来源：定位宣言v5手动微调，AI生成内容与创作者真实情况存在系统性偏差。

AI生成内容时，以下四类信息容易出现偏差，**必须由人工校验**：

### 19.1 事实精度优先于文学效果

AI倾向于做"文学加工"，将数字改得更夸张、时间改得更模糊。

**常见偏差**：
- 数字夸大：素材说"几天"，AI写成"两周"
- 结果弱化：素材说"降到了近乎为零"，AI写成"降到了五十多"（保留残余数字显得虎头蛇尾）

**校验规则**：所有涉及具体数字、时间跨度的表述，必须与素材原文或创作者确认后的一致。

### 19.2 措辞必须与最新定位保持同步

定位报告更新后，文章中的受众描述、核心主张等措辞不会自动同步。

**常见偏差**：
- 受众描述滞后：定位已从"有项目经验"扩展到"有实践经验"，但文章仍用"项目经验"
- 核心主张滞后：定位报告新增了能力维度，但文章未体现

**校验规则**：每次创作前，必须读取最新版作者事实档案§1（受众定位）和定位报告核心主张，确保措辞一致。

### 19.3 平台细节需人工确认

AI无法知道创作者实际在哪个平台发布内容。

**常见偏差**：
- 平台名称错误：实际发在小红书，AI写成"公众号"
- 平台特性不匹配：小红书适合图文+短标题，AI按公众号长文风格写

**校验规则**：涉及具体平台名称、平台特性适配的内容，必须由人工确认。

### 19.4 时间锚点需基于发文时间计算

AI生成的时间表达基于训练数据或上下文推断，容易出错。

**常见偏差**：
- 相对时间错误：2025年冬天的经历，2026年发文应为"前年"，AI写成"去年"
- 绝对时间错误：数据标注"截至2025年5月"，实际应为"截至2026年5月"

**校验规则**：所有相对时间（去年/前年/今年）和绝对时间（年月）表述，必须基于实际发文日期计算和校验。

## §19.5 引号使用精细化（AI写作架构文章经验沉淀）

> 来源：AI写作架构文章（20260526）人工微调——AI倾向于两种极端（过度使用或一刀切删除），需要精细规则。

AI倾向于两种极端：要么过度使用引号（每句话都包），要么一刀切全部删除。

**精细规则**：
- **保留引号**：①学术概念首次定义句（如"心理学管这叫「认知卸载」"）；②直接引用原话（如"Sam Altman说「搞砸了」"）；③专有名词首次出现
- **删除引号**：①转述他人意思（如"Sam Altman承认搞砸了"——不是原话）；②同一术语第二次及以后提及；③可用顿号或直接叙述替代的
- **禁止混用**：全文统一使用「」，不混用""

## §19.6 措辞与文章主旨一致性（AI写作架构文章经验沉淀）

> 来源：AI写作架构文章（20260526）人工微调——AI生成的措辞可能无意中与文章核心论点矛盾。

**常见偏差**：
- 文章主旨是"人机协作中创意源于人"，但措辞写成"AI帮你更好地思考"（暗示AI代替人思考）
- 文章主旨是"系统化思维"，但措辞写成"智能工厂"（暗示模板化、千篇一律）

**校验规则**：检查全文关键措辞是否与文章核心论点/定位声明一致。如有矛盾，调整为与主旨一致的表述。

**W1自检项**：
- [ ] 文中所有具体数字已与素材原文或创作者确认
- [ ] 受众描述与最新定位报告一致
- [ ] 涉及平台名称的内容已人工确认
- [ ] 所有时间表述已基于发文日期校验

## §20 开头故事技巧（Dan Koe评价经验沉淀）

> 来源：定位宣言v5 Dan Koe评价——开头故事获得满分。

### 20.1 画面感数据

用具体数字创造画面，而非模糊描述。

```
❌ 错误：代码库里有很多bug，我花了好几天才修完。
✅ 正确：代码库里堆着六百多个bug，没人知道从哪下手。
         我花了整整两天，和AI搭档一起排查、定位、修复。
```

**原则**：数字越具体，画面感越强。"六百多个"比"很多"强10倍。

### 20.2 痛点精准击中

描述"失去感"而非"获得感"。读者对"失去"的共鸣远强于"获得"。

```
❌ 错误：这个项目让我学到了很多东西，成长很大。
✅ 正确：然后这些排查经验就像水一样流走了。
         除了代码库里几行commit记录，好像什么都没剩下。
```

**原则**：开头故事的转折点必须是"失去感"——经验流失、时间浪费、机会错失。

### 20.3 共鸣建立

使用"你懂的""那种感受"等短语直接与读者对话，拉近距离。

```
✅ 正确：你懂的，那种感受。一个项目结束了，数据很漂亮，
         老板很满意。然后呢？
```

**原则**：在描述完痛点后，用"你懂的"或"那种感受"确认读者的共鸣。

### 20.4 真实感营造

适当自嘲，建立信任。完美人设不如真实人设。

```
✅ 正确：我不是什么写作大师。事实上，几个月前
         我还在为一篇几百字的文章发愁。
```

**原则**：在展示能力之前，先展示脆弱。自嘲→成长→能力，是建立信任的最短路径。

**W1自检项**：
- [ ] 开头故事是否使用了具体数字（非模糊描述）
- [ ] 开头故事的转折点是否为"失去感"
- [ ] 是否使用了"你懂的"或类似短语建立共鸣
- [ ] 是否在展示能力前先展示了真实/脆弱

## §21 结尾强化规范（Dan Koe评价经验沉淀）

> 来源：定位宣言v5 Dan Koe评价——结尾"下次再见"过于随意，建议用金句收束。

### 21.1 禁止随意告别

以下结尾方式**禁止使用**：
- "下次再见"
- "敬请期待"
- "感谢阅读"
- 任何无信息量的客套话

### 21.2 金句收束

结尾最后一段必须是加粗金句或核心观点重申，为读者留下记忆锚点。

```
❌ 错误：以上，如果觉得有点启发，随手点个赞吧。我们，下次再见。
✅ 正确：**毕竟，在这个AI时代，会做事的人不少。
         但能把事说清楚、能建立影响力的人，还是稀缺资源。**
```

### 21.3 CTA整合

CTA（行动号召）必须与金句整合，避免重复表述。

**推荐结构**：
```
点赞邀请 → 金句收束（加粗） → 行动指引（CTA）
```

```
❌ 错误（重复）：如果你也有实践经验但不知道怎么分享，评论区聊聊。
         ...（金句）...
         如果你也有实践经验但不知道怎么分享，我们聊聊。
✅ 正确（整合）：**金句收束。**
         如果你也有实践经验但不知道怎么分享，评论区聊聊。
```

### 21.4 结尾结构模板

```
[过渡段] 如果看到这里，可能也在想类似的问题...（场景代入）
[金句段] **核心观点重申。**（加粗，独立段落）
[CTA段]  如果你也...，评论区聊聊 / 私信我「关键词」获取XX。（具体行动指引）
```

**扩展模板（独立文章+系列引流）**：
```
[过渡段] 场景代入...
[分隔线] ---
[金句段] **核心观点重申。**（加粗，独立段落）
[延伸阅读] > **延伸阅读**：如果你想了解...，可以看看这个系列：[链接]
[身份介绍] 我是XX，一个...的人。
```

> 来源：AI写作架构文章（20260526）人工微调——独立文章结尾需要金句收束+延伸阅读引流+身份介绍。

**W1自检项**：
- [ ] 结尾是否使用了随意告别语（"下次再见"等）
- [ ] 最后一段是否为金句或核心观点
- [ ] CTA是否只出现一次且与金句无重复
- [ ] 结尾结构是否符合：过渡→金句→CTA（或扩展模板：金句→延伸阅读→身份介绍）
