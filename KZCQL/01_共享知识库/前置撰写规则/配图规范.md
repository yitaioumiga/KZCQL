# 配图规范

> 版本：v1.0 | 生效日期：2026-05-14 | 依据：SG-2026-0514-02 联合督查报告

---

## 1. 目录结构规范

所有文章配图统一存放于以下目录：

```
03_文章产出/
├── images/
│   ├── 01/          # 第1篇文章配图
│   ├── 02/          # 第2篇文章配图
│   ├── 03/          # 第3篇文章配图
│   └── ...
├── 配图/            # 历史归档目录（已废弃，不再使用）
└── 稿件/
```

- 每篇文章对应一个以文章编号命名的子目录（两位数字，不足补零）
- 原始 `配图/` 目录保留作为历史归档，不再新增文件

## 2. 配图命名规范

### 格式

```
{文章编号}_{序号}_{主题关键词}.jpg
```

### 示例

| 原始文件名 | 规范文件名 | 说明 |
|-----------|-----------|------|
| 第3篇_封面_先稳后攻.jpg | 03_01_封面先稳后攻.jpg | 第3篇第1张，封面 |
| 第3篇_资产配置天平.jpg | 03_02_资产配置天平.jpg | 第3篇第2张 |
| 01_封面.jpg | 01_01_封面.jpg | 第1篇第1张，封面 |

### 命名规则

- **文章编号**：两位数字，与文章目录编号一致（01、02、03...）
- **序号**：两位数字，从01开始递增
- **主题关键词**：简短描述图片内容，使用中文，不含特殊字符
- **文件扩展名**：统一使用 `.jpg`
- **禁止**：空格、特殊符号、中文标点

## 3. 配图引用路径规范

### Markdown 中的引用

使用相对路径，基于 Markdown 文件所在目录（`稿件/`）：

```markdown
![封面](../images/03/03_01_封面先稳后攻.jpg)
```

### 路径规则

- 从 `稿件/` 目录出发，使用 `../images/` 进入配图目录
- 禁止使用绝对路径（如 `/workspace/...`）
- 禁止使用包含中文的路径

## 4. 配图质量标准

| 项目 | 要求 |
|------|------|
| 分辨率 | 1280 x 720 px（最低标准） |
| 宽高比 | 16:9 |
| 格式 | JPG |
| 文件大小 | 单张不超过 2MB |
| 水印 | 禁止 AI 生成水印 |
| 内容 | 与文章主题相关，不得使用无意义装饰图 |

## 5. 配图数量规范

| 文章类型 | 配图数量 | 说明 |
|---------|---------|------|
| 系列文章 | 3-5 张/篇 | 封面1张 + 正文配图2-4张 |
| 封面图 | 必须有 | 每篇文章至少1张封面图 |

## 6. 历史文件清理规则

### 废弃配图处理

- 废弃配图在文件名前添加 `[废弃]` 前缀
- 示例：`cover.jpg` -> `[废弃]_cover.jpg`
- 废弃配图保留 30 天，到期后由人工确认删除
- 废弃配图不参与 Markdown 引用

### 废弃判定标准

- 旧版命名的配图（如 `cover.jpg`、`illustration1.jpg`）
- 重复版本（如 `_v2` 后缀的旧版）
- 不再被任何 Markdown 文件引用的配图

## 7. 新增配图流程

1. 按照命名规范重命名配图文件
2. 放入对应的 `images/{文章编号}/` 目录
3. 在 Markdown 中使用相对路径引用
4. 检查配图是否符合质量标准
