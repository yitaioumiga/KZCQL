# 通用归档规则

> **适用范围**：所有子 Agent（W1/W2/R1/R2/R3/R4/I1/A1/E1）
> **强制等级**：必须遵守
> **最后更新**：2026-04-24

---

## 一、核心原则

**每个 Agent 的每次产出都必须归档。** 归档是产出的最后一步，未归档的产出视为未完成。

---

## 二、归档目录结构

所有 Agent 的产出物统一归档到 `04_工作区/` 下的对应子目录：

```
04_工作区/
├── 产出归档/              ← 已完成文章的完整归档（按日期分组）
│   └── {YYYYMMDD}_{文章关键词}/
│       ├── 稿件/          ← 文章各版本（.md + .docx）
│       ├── images/        ← 该文章的所有配图（统一命名为images/）
│       ├── 评审/          ← R1/R2/R3/R4 报告
│       └── 素材/          ← 原始素材
├── 督察归档/              ← I1 督察报告（保持不变）
│   ├── 定向审查/
│   ├── 事故审查/
│   └── 标准抽检/
├── 架构归档/              ← A1 架构审查报告（保持不变）
└── 原始材料/              ← 通用原始素材（保持不变）
```

---

## 三、文件命名规范

### 3.1 评审报告

格式：`{文章关键词}_{Agent代号}_第{N}轮.{ext}`

示例：
- `AI代码质量_R2_第1轮.md`
- `AI代码质量_R3_第2轮.md`
- `AI代码质量_R4_最终评级.md`

**归档要求**：报告完成后**必须**保存至 `04_工作区/产出归档/{YYYYMMDD}_{文章关键词}/评审/`，命名格式 `{文章关键词}_R2_第{N}轮.md`。详见 `01_共享知识库/通用归档规则.md`。

归档位置：`04_工作区/产出归档/{YYYYMMDD}_{文章关键词}/评审/`（注：评审报告归档路径已从"评审归档/"统一调整至各文章产出目录下的"评审/"子目录）

### 3.2 督察/架构报告

格式：`{报告类型}_{日期}_{序号}.{ext}`

示例：
- `定向审查_20260423_01.md`
- `事故审查_INC-20260423-01.md`
- `架构审查_20260423_01.md`

### 3.3 文章稿件

格式：`{文章关键词}_v{N}.{ext}`

示例：
- `AI代码质量_v1.md`
- `AI代码质量_v2.md`

归档位置：`04_工作区/产出归档/{YYYYMMDD}_{文章关键词}/稿件/`

## 四、归档时机

| Agent | 归档时机 | 归档内容 |
|-------|----------|----------|
| W1 | 初稿完成时 | 初稿 .md + 配图 |
| W2 | 每轮修改完成时 | 修改后稿件 + 修改说明 |
| R1 | 核查完成时 | 事实核查报告 |
| R2 | 评审完成时 | 全量评审报告 |
| R3 | 差异评审完成时 | 差异评审报告 |
| R4 | 评级完成时 | 评级报告 |
| I1 | 审查完成时 | 审查报告（按类型归档） |
| A1 | 审查完成时 | 架构审查报告 |
| E1 | 规则修改完成时 | 规则变更记录（归档至 05_人类反馈与规则演进/） |

---

## 五、归档完成流程

文章完成全部评审流程（R4 最终评级为 S/A 或用户确认交付）后，必须执行归档完成流程：

1. 创建产出归档目录：`04_工作区/产出归档/{YYYYMMDD}_{文章关键词}/`
2. 将稿件移动到 `产出归档/{YYYYMMDD}_{文章关键词}/稿件/`
3. 将配图移动到 `产出归档/{YYYYMMDD}_{文章关键词}/images/`（统一使用images/命名）
4. 将评审报告移动到 `产出归档/{YYYYMMDD}_{文章关键词}/评审/`
5. 清理工作区中的临时文件

---

## 六、JSON 摘要要求

每个 Agent 的产出**必须**包含 JSON 摘要，用于快速检索和回溯：

```json
{
  "agent": "{Agent代号}",
  "article": "{文章关键词}",
  "round": {轮次},
  "timestamp": "{时间}",
  "file_path": "{归档文件路径}",
  "key_findings": ["{关键发现1}", "{关键发现2}"]
}
```

JSON 摘要附在 Markdown 报告末尾。
