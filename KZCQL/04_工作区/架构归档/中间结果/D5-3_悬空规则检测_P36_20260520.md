## 三、IMG-01b 验证命令准确性分析

### 3.1 当前验证命令

```bash
grep -c '图片：{' {文件}.md  # 结果必须为0
```

### 3.2 问题分析：无法捕获的占位符变体

根据前置撰写规则 §4.8.1，**严禁使用占位符格式**包括：

```markdown
❌ 图片：{封面——一个年轻人坐在书桌前...}
❌ {图片：章节插图——三个台阶...}
❌ 图片占位符：{...}
```

**当前命令 `grep -c '图片：{'` 的覆盖情况：**

| 占位符变体 | 示例 | 是否被捕获 | 风险 |
|------------|------|------------|------|
| 标准格式 | `图片：{封面...}` | ✅ 是 | 无 |
| 带大括号前缀 | `{图片：章节插图...}` | ❌ **否** | **漏检** |
| 带"占位符"后缀 | `图片占位符：{...}` | ❌ **否** | **漏检** |
| 英文冒号 | `图片:{...}` | ❌ **否** | **漏检** |
| 空格变体 | `图片： {..}`（冒号后有空格） | ❌ **否** | **漏检** |

### 3.3 建议的增强验证命令

```bash
# 建议命令1：捕获所有"图片"+冒号+大括号变体
grep -cE '(图片|占位符).*[:：].*\{' {文件}.md  # 必须为0

# 建议命令2：捕获所有大括号内的"图片"关键词
grep -c '\{[^}]*图片[^}]*\}' {文件}.md  # 必须为0

# 建议命令3：组合验证（推荐）
grep -cE '(图片[:：]|\{图片|图片占位符)' {文件}.md  # 必须为0
```

## 四、自检三问 Enforcement 机制分析

### 4.1 规则原文（主Agent编排指南 §5.5）

主Agent在标记"全部通过"前，必须回答：

1. **我是否对每个检查项使用了指定的验证工具？**
2. **我是否记录了每个检查项的工具输出？**
3. **我是否根据工具输出（而非主观判断）做出通过/不通过判定？**

**任一答案为"否" → 重新执行检查**

### 4.2 Enforcement 机制缺口

| 要求 | 当前机制 | 缺口分析 | 风险等级 |
|------|----------|----------|----------|
| 强制回答三问 | 文本描述要求 | **无技术强制手段**，依赖主Agent自觉遵守 | P1 |
| 答案记录 | 建议记录到"检查执行日志" | **无强制归档要求**，日志可能缺失 | P1 |
| "否"触发重检 | 文本描述 | **无自动化阻断机制**，主Agent可自行标记通过 | P0 |
| 答案验证 | 无 | **无第三方验证机制**，无法确认答案真实性 | P2 |

### 4.3 建议的 Enforcement 增强

```markdown
## 建议：自检三问强制化机制

1. **工具化强制**：将三问嵌入检查执行日志模板，未回答则无法提交
2. **结构化记录**：要求以特定格式记录答案
   ```markdown
   ### 自检三问答案
   - [x] 问题1：是 | 证据：{工具输出截图/引用}
   - [x] 问题2：是 | 证据：{日志记录位置}
   - [x] 问题3：是 | 证据：{判定依据}
   ```
3. **质量门检查**：在"交付保障检查"中增加三问验证子项
4. **违规后果**：未完整回答三问 → 检查自动判定为"未完成"
```

## 五、规则-执行缺口详细分析

### 5.1 缺口分类统计

| 缺口类型 | 数量 | 占比 | 示例 |
|----------|------|------|------|
| 完全悬空（无工具验证） | 6项 | 42.9% | IMG-03, IMG-05, IMG-06, REV-01, CLN-01, ITR-01 |
| 部分悬空（工具不完整） | 3项 | 21.4% | IMG-02, FMT-01, FMT-02 |
| 命令缺陷（覆盖不全） | 1项 | 7.1% | IMG-01b |
| Enforcement 缺失 | 1项 | 7.1% | 自检三问 |
| 已落地 | 3项 | 21.4% | IMG-01, IMG-01b(部分), ARC-01~04 |

### 5.2 关键缺口详解

#### 缺口1：目视检查的可靠性问题

**涉及检查项**：IMG-03, IMG-05, IMG-06

**问题描述**：
- "目视检查"依赖主Agent的主观判断
- 不同主Agent对"水印可见性""质量达标"的判断标准不一致
- 无法生成可审计的验证证据

**事故关联**：
> INC-2026-0520-图片格式：14项检查全部通过但交付物不合格

**根因分析**：
- IMG-03（水印检查）依赖目视，可能漏检微小水印
- IMG-05（分辨率/质量）依赖目视，无法精确判断像素级问题

#### 缺口2：IMG-01b 命令覆盖不全

**问题描述**：
- 当前命令 `grep -c '图片：{'` 仅捕获标准格式
- 变体格式 `{图片：...}` 和 `图片占位符：{...}` 可能漏检

**风险**：占位符格式进入交付流程 → 生产事故

#### 缺口3：自检三问无强制力

**问题描述**：
- 三问是"软要求"，非"硬阻断"
- 主Agent可在未回答的情况下标记"全部通过"

**风险**：自我欺骗式检查通过 → 交付物不合格

1. **P36补丁后仍存在严重悬空规则**：14项检查中，6项（42.9%）完全悬空，3项（21.4%）部分悬空

**P36补丁仅完成了部分修复**：
- ✅ 已新增强制工具验证清单框架
- ✅ 已明确IMG-01/IMG-01b/ARC系列的验证命令
- ❌ 目视检查项仍未解决
- ❌ 自检三问仍未强制化

**建议**：执行P0和P1级别的修复建议，将评分提升至B级（≥10分）以上。
