### V-01 虚构事实 | 未触发

逐条验证文章中的关键事实声明：

#### 事实1: Cursor 的 .cursorrules 配置方式
- **文章声明**: "旧版是直接在项目根目录放一个 .cursorrules 文件...新版更灵活一些，支持在 .cursor/rules/ 目录下放多个 .mdc 文件"
- **验证结果**: **准确**。多个来源确认 .cursorrules 为旧版（legacy），.cursor/rules/ 下 .mdc 文件为新版标准格式。
- **来源**: cursor.fan, apidog.com, GitHub awesome-cursor-rules-mdc

#### 事实2: Claude Code 的 CLAUDE.md 配置方式
- **文章声明**: "CLAUDE.md，放在项目根目录就行，Markdown 格式...支持全局配置，你可以在 ~/.claude/CLAUDE.md 放一份全局规则"
- **验证结果**: **准确**。Claude Code 官方文档确认支持项目根目录 CLAUDE.md 和 ~/.claude/CLAUDE.md 全局配置。
- **来源**: code.claude.com/docs/en/settings, claude.com/blog/using-claude-md-files

#### 事实3: Claude Code "只支持单文件"
- **文章声明**: "Claude Code 只支持单文件，不像 Cursor 新版那样可以拆成多个文件按功能分类"
- **验证结果**: **不准确，但不构成虚构事实**。Claude Code 实际上已支持 `.claude/rules/` 目录下的多文件规则系统（支持 glob 模式匹配），且支持子目录 CLAUDE.md 层级结构。但这一功能相对较新且社区认知度不高，文章作者可能基于自身使用经验做出了过时描述。
- **严重程度评估**: 属于事实偏差/过时描述，但非"编造"。Claude Code 的核心配置文件确实是 CLAUDE.md（单文件），`.claude/rules/` 是扩展功能。文章说"只支持单文件"虽不完全准确，但描述的是主流使用方式。
- **来源**: ytrofr.github.io/claude-code-guide, github.com/anthropics/claude-code/issues

#### 事实4: Copilot 的 .github/copilot-instructions.md 配置方式
- **文章声明**: "配置文件叫 .github/copilot-instructions.md...支持 YAML frontmatter...指定这条规则适用于哪些文件"
- **验证结果**: **准确**。GitHub 官方文档确认文件路径为 `.github/copilot-instructions.md`，且支持 YAML frontmatter 中的 `applyTo` 字段进行文件匹配。
- **来源**: docs.github.com/en/copilot/how-tos/configure-custom-instructions, 掘金多篇文章

#### 事实5: MCP 发布时间和架构
- **文章声明**: "Anthropic 在 2024 年底推出的一个开放协议...架构分三层，Host 是宿主，Client 是客户端，Server 是服务器"
- **验证结果**: **准确**。MCP 于 2024 年 11 月由 Anthropic 发布。架构确实是 Host/Client/Server 三层。官方规范文档和 IETF 报告均确认。
- **来源**: IETF interim-2025-nmrg-02 报告, modelcontextprotocol.io, model-context-protocol.github.io

#### 事实6: USB 1.0 发布时间
- **文章声明**: "1996 年 USB 1.0 发布"
- **验证结果**: **准确**。USB 1.0 于 1996 年 9 月发布。
- **来源**: omniscient.wiki, systemtek.co.uk

#### 事实7: Soul Spec 项目存在性

- **验证结果**: **准确**。soulspec.org 确认项目存在，支持 SOUL.md 和 .soul/ 目录结构。
- **来源**: soulspec.org, github.com/clawsouls, npmjs.com/package/clawsouls

#### 事实8: Soul Spec 社区注册表 "81 多个预置配置"

- **验证结果**: **基本准确**。ClawSouls 官方描述为 "80+ curated personas" 和 "80+ souls available"，文章说"81 多个"在合理范围内。
- **来源**: github.com/clawsouls, toolify.ai, npmjs.com/package/clawsouls

#### 事实9: SoulScan 安全扫描工具

#### 事实10: AGENTS.md 多 Agent 协作
