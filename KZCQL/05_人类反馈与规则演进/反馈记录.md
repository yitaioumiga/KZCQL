# 人类反馈记录

## 用途

本文档用于记录人类审阅者在终审阶段提供的所有反馈，是规则演进和持续改进的核心数据来源。每条反馈都应被完整记录、分类处理、闭环追踪。

---

## 反馈转化规则

### 自动触发规则

| 触发条件 | 处理动作 | 说明 |
|----------|----------|------|
| 同类反馈出现 **3次** | 自动触发规则变更评估 | 由 E1 规则演进 Agent 分析是否需要修改规则 |
| 反馈类型为 **"规则反馈"** | 直接触发 A1 架构审查 | 跳过计数，立即进入架构审查流程 |

### 反馈处理状态流转规则

```
待处理 → 处理中 → 已完成
   ↓        ↓         ↓
 可关闭   可关闭    归档保存
```

**状态定义**：
- **待处理**：反馈已记录，尚未分配处理人
- **处理中**：已分配处理人，正在执行修改或规则变更
- **已完成**：反馈已处理完毕，结果已验证并归档
- **已关闭**：反馈因不适用、重复、或用户撤回等原因关闭

**流转规则**：
1. 新反馈默认进入「待处理」状态
2. 分配处理人后流转至「处理中」
3. 处理完成并验证后流转至「已完成」
4. 任意状态均可流转至「已关闭」（需注明关闭原因）

## 反馈类型说明

| 类型 | 说明 | 处理方式 |
|------|------|----------|
| 通过 | 文章质量达标，无需修改 | 直接交付 |
| 修改意见 | 对文章内容、结构、表达等提出修改建议 | 返回 W2 迭代修改 |
| 规则反馈 | 对评审规则、写作规则等提出调整建议 | 触发 E1 规则修改流程 |
| 混合 | 同时包含修改意见和规则反馈 | 先执行修改，再执行规则变更 |

---

## 处理状态说明

| 状态 | 含义 |
|------|------|
| 待处理 | 反馈已记录，尚未开始处理 |
| 处理中 | 正在执行对应的修改或规则变更 |
| 已完成 | 反馈已处理完毕，结果已验证 |
| 已关闭 | 反馈因某种原因关闭（如不适用、重复等） |

## 结构化反馈模板

> 人类终审时请按以下模板填写反馈，确保反馈可被系统化分析和转化为规则变更。

### 模板格式

```json
{
  "feedback_id": "FB-{日期}-{序号}",
  "article": "文章关键词",
  "feedback_date": "YYYY-MM-DD",
  "feedback_type": "满意 / 修改 / 重写",
  "overall_rating": "S / A / B / C / D / F",
  "satisfied_dimensions": ["D1", "D3", "D7"],
  "unsatisfied_dimensions": [
    {
      "dimension": "D2",
      "issue": "具体问题描述",
      "severity": "高 / 中 / 低",
      "suggestion": "具体修改建议"
    }
  ],
  "trigger_rule_change": true,
  "rule_change_suggestion": "建议新增/修改的规则描述（如不触发规则变更则为空）",
  "free_text": "其他补充意见（自由文本）"
}
```

### 填写说明

| 字段 | 说明 | 示例 |
|------|------|------|
| feedback_type | 三选一：满意/修改/重写 | "修改" |
| overall_rating | 与 R4 评级对齐 | "B" |
| satisfied_dimensions | 表现好的维度（D1-D10） | ["D1", "D7"] |
| unsatisfied_dimensions | 需要改进的维度及具体问题 | 见上方格式 |
| trigger_rule_change | 此反馈是否应触发规则变更 | true/false |
| rule_change_suggestion | 如触发，建议的规则变更内容 | "R2应在D7中增加对深色背景配图的检查" |
