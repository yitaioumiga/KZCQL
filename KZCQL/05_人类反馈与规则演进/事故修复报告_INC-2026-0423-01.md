## 四、修复后的防护链路

修复前后的对比：

```
【修复前】生成 → [无验证] → R2盲评 → R3跳过 → R4采信 → 质量门不拦 → 交付

【修复后】生成 → ImageCheck(Read逐张) → R1事实核查 → R2(Read逐张+逐图表格)
       → R3(配图复检) → R4(异常标记) → 质量门(配图检查) → 一票否决(V-16/V-17)
       → 交付
```

## 五、同类事故预防保障

### 5.1 强制执行机制

| 机制 | 说明 |
|------|------|
| **Read 强制指令** | R2/R3 评审 D7 时必须用 Read 工具逐一查看图片，否则 D7 评分为无效 |
| **逐图检查表格** | D7 评语必须包含逐图检查结果表格，无表格则 R4 标记为「需复检」 |
| **一票否决** | V-16（水印）和 V-17（乱码）为致命否决项，触发即退回 |
| **强制声明** | §4.5 防文字声明、§4.6 水印处理、§5.2 渲染方案均使用「必须」「严禁」强约束词 |
| **生产事故定性** | §4.6 明确"跳过此步骤视为生产事故" |

### 5.2 职责归属清晰化

| 角色 | 配图职责 | 来源 |
|------|----------|------|
| W1 初稿撰写 | 生成配图 + 防文字声明 + 水印裁剪 + Read 验证 | 初稿撰写Agent.md Step 9 |
| 主Agent | ImageCheck 流程节点 + 质量门配图检查 | 编排指南 + 编排流程图 |
| R2 全量评审 | Read 逐一查看图片 + 逐图检查结果表格 | 全量评审Agent.md D7 特殊流程 |
| R3 差异评审 | 配图复检（修改涉及 D7 时） | 差异评审Agent.md D7 特殊规则 |
| R4 评级判定 | 异常评分标记 + 需复检机制 | 评级判定Agent.md Step 2.5 |

### 5.3 修复后的完整配图工作流

```
1. W1 生成文艺插图
   ├── GenerateImage + 防文字声明（§4.5 强制追加）
   ├── Pillow 裁剪水印（§4.6 强制执行）
   └── Read 验证水印已去除

2. W1 生成架构图
   ├── Playwright/wkhtmltoimage HTML 渲染（§5.2 严禁 GenerateImage）
   └── Read 验证中文清晰可读

3. ImageCheck（流程节点）
   ├── Read 逐张检查所有图片
   ├── 不通过 → 重新生成 → 再次验证
   └── 通过 → 进入 R1

4. R2 评审 D7
   ├── Read 逐一打开每张图片
   ├── 逐图检查结果表格
   └── 如发现水印/乱码 → 触发 V-16/V-17 否决

5. R3 差异评审
   ├── 修改涉及 D7 → Read 查看修改后图片
   └── 新问题检测 → 配图质量检查

6. R4 评级判定
   ├── Step 2.5 异常评分标记
   └── D7 无逐图表格 → 标记「需复检」

7. 质量门
   ├── 配图已生成且通过 Read 验证
   └── 架构图使用 HTML 渲染

8. 一票否决
   ├── V-16：配图水印 → 致命
   └── V-17：架构图乱码 → 致命
```
