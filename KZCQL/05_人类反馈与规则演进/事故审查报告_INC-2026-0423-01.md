# KZCQL 配图生产事故审查报告

> **事故编号**：INC-2026-0423-01
> **事故等级**：P0（生产事故 — 不合格交付物通过评审）
> **审查时间**：2026-04-23
> **审查范围**：KZCQL 全架构（编排层、评审层、规则层、Agent规范层）

---

## 一、事故概述

文章《AI写的代码靠不靠谱谁说了算》在 KZCQL 工作流中完成了 W1→L1-L4→R1→R2→W2→R3→R4 全流程，最终获得 88/100（A级）评级，通过评审门槛。

**实际情况**：5 张配图中 4 张存在严重质量问题：
- 4 张带有 "TRAE AI 生成" 水印
- 3 张中文文字乱码/伪文（如"原則則圖圖""所荏的发进化图图"）
- 3 张英文伪文（如"Gromiration""Chockoss""Manuning"）
- 架构图使用 GenerateImage 渲染（违反 §5.2 强制 HTML 渲染规则）

**事故性质**：不合格交付物通过了全部评审环节，评级虚高至少 6 分（D7 虚评 9 分，实际应为 2-3 分）。

---

## 二、根因分析

### 2.1 根因一：三层验证全部缺失（致命）

从配图生成到最终交付，整条链路上没有任何一个环节强制要求用 Read 工具实际查看图片内容：

```
生成配图（GenerateImage）
    ↓
[断层 1] 生成后无 Read 验证 → 水印、乱码未被发现
    ↓
R2 全量评审（D7 配图质量）
    ↓
[断层 2] R2 输入只有 Markdown 文本，无法看到图片 → D7 盲评
    ↓
R3 差异评审
    ↓
[断层 3] R3 新问题检测清单无配图项 → 修改后图片未复检
    ↓
R4 评级判定
    ↓
[断层 4] R4 "只汇总不创造"，被禁止验证上游评分 → 虚假高分穿透
    ↓
质量门检查
    ↓
[断层 5] 6 项质量门无配图相关项 → 不拦截
    ↓
交付（88分，A级）
```

**结论**：配图质量的保障完全依赖于生成时的"一次性成功"，没有任何事后验证机制。

### 2.2 根因二：规则存在但执行无保障（严重）

| 规则条款 | 规则位置 | 规定内容 | 为什么没执行 |
|----------|----------|----------|-------------|
| §4.3 禁止文字 | 前置撰写规则 L265 | "**禁止**出现任何文字" | GenerateImage 的 prompt 未追加防文字声明 |
| §4.5 防文字声明 | 前置撰写规则 L281 | "**务必**加上防文字声明" | "务必"是弱约束词，且未被注入 Agent 工作流 |
| §4.6 水印处理 | 前置撰写规则 L289 | 用 Pillow 裁掉水印 | 被设计为"方法参考"而非"必执行步骤" |
| §5.2 渲染方案 | 前置撰写规则 L347 | "GenerateImage 对中文渲染不可控" | 使用"仅限"而非"禁止"，Agent 可合理选择 |
| Step 8 去除水印 | 编排指南 L264 | "确认水印已完全去除" | 描述性文字，无强制执行机制 |
| 发布前检查清单 | 发布前检查清单 L20-26 | 8 项配图检查 | **未被任何流程引用**，悬空文件 |

**核心问题**：规则文件中大量使用"务必""首选""经验"等弱约束词，缺乏"必须""禁止"等强约束词。违反后果缺失或过软（铁律 15 的后果是描述性的，非惩罚性的）。

### 2.3 根因三：配图职责归属矛盾（严重）

| 文件 | 声称的负责方 | 实际情况 |
|------|-------------|----------|
| 主Agent编排指南 L286 | "W1 负责" | W1 规范文件（313行）中**完全没有配图职责** |
| 智能体注册表 L250 | 主Agent微调路由 | 主Agent被禁止直接操作内容（L20） |
| W1 初稿撰写Agent | 无 | 8步流程、四层自检、禁止事项中均无配图 |
| W2 迭代修改Agent | 无 | 修改策略表中无配图相关条目 |

**结论**：配图生成是一个"人人都以为有人做，但没人实际做"的职责真空。

### 2.4 根因四：评审体系对 D7 盲评（致命）

| 评审 Agent | D7 评审方式 | 问题 |
|-----------|------------|------|
| R2 全量评审 | 只看 Markdown 文本中的 `![](path)` 引用 | 无法看到图片实际内容，D7 评分实质是"形式审查" |
| R3 差异评审 | 只看 Markdown diff | 新问题检测清单（5项）中无配图质量项 |
| R4 评级判定 | 只汇总 R2/R3 分数 | 被规则禁止验证上游评分（"禁止修改分数"） |

**D7 评语模板的设计缺陷**：要求引用 `{文章中的具体段落}`，但图片不是段落。这个模板引导 Agent 去引用 `![描述](images/xxx.jpg)` 而非实际图片内容。

### 2.5 根因五：否决机制不覆盖配图（中等）

15 项一票否决项（V-01 到 V-15）中没有图片相关项。图片有严重水印、图片完全缺失等问题最多扣 10 分（D7 满分），不会触发否决退回。对比：标题超过 20 字（V-08）可否决，但配图完全缺失却不能。

## 五、审查结论

本次配图生产事故不是单点故障，而是 **KZCQL 架构的系统性验证缺失**：

1. **规则层**：规范写得好（§4/§5 详细且合理），但强制性不足，且未被转化为 Agent 可执行的步骤
2. **编排层**：配图环节在流程图中不可见，质量门不覆盖配图，发布前检查清单悬空
3. **评审层**：R2/R3/R4 三层评审全部无法实际查看图片，D7 评审实质是盲评
4. **Agent 层**：配图职责归属矛盾，W1 不知道自己需要生成配图

**一句话总结**：规则存在但执行无保障，评审存在但验证无手段，职责存在但归属不清晰。
