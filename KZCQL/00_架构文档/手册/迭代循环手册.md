## 阶段四：迭代循环

**目标**：通过多轮「修改 → 评审」循环，持续提升文章质量。

---

## 4.1 迭代流程

```
迭代循环（每轮）
│
├── 4.1.1 W2 迭代修改
│   ├── 输入：当前版本文章 + 评审报告 + W2 修改规范
│   ├── 输出：修改后文章（版本号递增）+ 修改说明
│   └── 版本号规则：v0.1 → v0.2 → v0.3 → ...
│
├── 4.1.1b R1 复检（当前置 R1=FAIL 时必须执行）
│   ├── 触发条件：前置 R1 判定为 FAIL
│   ├── 输入：W2 修改后文章 + 前置 R1 报告中的触发项清单
│   ├── 核查范围：仅核查前置 R1 报告中的触发项是否已解除
│   ├── 输出：R1 复检报告（PASS/FAIL + 逐项确认）
│   ├── R1 复检=PASS → 执行 4.1.2 差异评审
│   └── R1 复检=FAIL → 返回 4.1.1 W2 重新修改
│
├── 4.1.2 差异评审（第2轮起）
│   ├── 第2轮：R3 差异评审（对比 v0.1 vs v0.2）
│   ├── 第3轮：R3 差异评审（对比 v0.2 vs v0.3）
│   └── ...以此类推
│
├── 4.1.2b R3 阻断处理（如 R3 标记"需R1复检"）
│   ├── 触发条件：R3 差异评审中标记"需R1复检"并阻断流程
│   ├── 处理方式：回到 R1 复检（4.1.1b），仅核查 R3 标记的问题项
│   ├── R1 复检=PASS → 重新执行 4.1.2 差异评审
│   └── R1 复检=FAIL → 返回 4.1.1 W2 重新修改
│
├── 4.1.3 R2 风格评审（如 R1=PASS）
│   └── 确保修改未破坏风格一致性
│
└── 4.1.4 R4 综合评级（硬性阻断点 — 不可跳过）
    ├── R4 是每次迭代循环的**唯一出口**
    ├── 无 R4 评级报告 → 不得进入任何后续环节
    └── 主代理不得基于 R3 报告的"下一步建议"跳过 R4
```

---

## 4.2 迭代退出条件（3 级优先级）

| 优先级 | 退出条件 | 说明 | 处理方式 |
|--------|----------|------|----------|
| **P0（最高）** | 达到 A 级或以上 | 文章质量已达标 | 进入人工终审 |
| **P1** | 连续 2 轮评级无提升 | 迭代已收敛，继续修改无意义 | 进入人工终审，附带说明 |
| **P2（兜底）** | 满 5 轮迭代 | 防止无限循环 | 强制退出，进入人工终审，附带说明 |

### 退出条件检查流程

```
R4 评级输出
  │
  ├── 评级 >= A（8.0）？
  │   └── 是 → P0 触发，退出迭代
  │
  ├── 本轮评级 == 上轮评级？
  │   └── 是 → 计数器 +1
  │       ├── 计数器 >= 2 → P1 触发，退出迭代
  │       └── 计数器 < 2 → 继续迭代
  │
  ├── 本轮评级 < 上轮评级？
  │   └── 是 → 重置计数器，继续迭代（可能需要调整修改策略）
  │
  └── 迭代轮次 >= 5？
      └── 是 → P2 触发，强制退出
```

### 特殊情况处理

| 情况 | 处理方式 |
|------|----------|
| 迭代中评级下降 | 重置「连续无提升」计数器，但增加总轮次计数。如果连续 2 轮下降，考虑返回 W1 重写 |
| 迭代中评级大幅下降（>1分） | 立即中止迭代，返回 W1 重写 |
| W2 修改引入新问题 | R3 标记「引入新问题」，本轮评级不升反降，计入迭代轮次 |

## 4.3 阶段四出口检查（SG-2026-0514-02补丁P9，强制执行）

> **事故编号**：SG-2026-0514-02
> **漏洞编号**：D-02
> **生效日期**：2026-05-14
> **触发事件**：第3篇v2在W2修改后直接跳过R2评审进入R4评级

W2 迭代修改完成后、进入下一轮评审前，主Agent**必须**执行以下出口检查：

```
阶段四出口检查流程
│
├── 1. 读取修改说明.md
│   └── 确认修改说明文件存在且内容完整
│
├── 2. 检查"修改后评审触发判定"章节
│   └── 定位修改说明中的触发判定清单
│
├── 3. 判定评审路径
│   ├── 存在任一触发条件勾选 → 强制进入R2全量评审（不得跳过）
│   ├── 无触发条件 → 确认修改说明中有"无需重评"标注及理由
│   └── 缺少"修改后评审触发判定"章节 → 视为流程违规，必须补写
│
└── 4. 阻断规则
    ├── 未执行出口检查 → 不得进入任何后续评审环节
    ├── 触发R2重评但跳过 → R4评级结果自动无效
    └── 缺少触发判定章节 → 返回W2补写，不得继续
```

### 检查清单

- [ ] 修改说明.md 已读取
- [ ] "修改后评审触发判定"章节存在
- [ ] 如有触发项 → 已安排R2全量评审
- [ ] 如无触发项 → 已确认"无需重评"标注及理由
- [ ] 未出现跳过R2直接进入R4的情况
