# W1 初稿撰写Agent 输入包模板

## 一、Agent 概述

- **代号**: W1
- **角色定位**: 写作组核心创作者，负责从零撰写文章初稿
- **核心职责**: 基于用户主题和调研素材，调用khazix-writer Skill撰写符合卡兹克风格的初稿

## 三、调用前验证清单

- [ ] 自身规范文件路径已包含在Task query中
- [ ] 前置撰写规则文件存在
- [ ] 用户主题非空
- [ ] 原始材料/主题描述已提供

- [ ] 输出包含完整的Markdown文章
- [ ] 文章包含4-7张配图引用
- [ ] 文章已执行四层自检（L1-L4）
- [ ] 文章已生成docx文件
- [ ] 文件已保存至正确的归档路径

## 五、错误处理

### 5.1 文件不存在

| 文件 | 处理方式 |
|------|----------|
| 自身规范文件 | **阻断**：终止调用，报告文件缺失 |
| 前置撰写规则 | **阻断**：终止调用，报告文件缺失 |
| 作者事实档案 | **降级**：如不涉及作者个人经历，可跳过；否则终止调用 |
| 系列文章写作规范 | **降级**：非系列文章时跳过 |

### 5.2 输出格式不符

| 问题 | 处理方式 |
|------|----------|
| 缺少必备章节 | 要求W1补充缺失章节 |
| 配图数量不足（<4张） | 要求W1补充配图 |
| 未生成docx文件 | 要求W1生成docx |
| 文件未保存至正确路径 | 主Agent手动移动文件 |

### 5.3 Agent执行失败

| 场景 | 处理方式 |
|------|----------|
| khazix-writer Skill调用失败 | 重试1次，仍失败则向用户报告 |
| 四层自检未通过 | 要求W1修复后重新自检 |
| 配图生成失败 | 要求W1重新生成失败配图 |

## 七、建议补充内容（主Agent自由裁量）

> **说明**：以下内容非强制要求，主Agent可根据具体工作场景判断是否需要补充。

### 7.1 场景化补充建议

| 场景 | 建议补充内容 | 补充原因 |
|------|-------------|----------|
| 技术类文章 | 相关技术文档/白皮书链接 | 确保技术细节准确 |
| 涉及数据 | 数据来源说明/统计口径 | 增强可信度 |
| 争议性话题 | 对立观点材料 | 确保论证平衡 |
| 时效性强 | 最新行业动态/新闻 | 确保内容新鲜度 |
| 跨领域主题 | 领域术语表 | 帮助W1理解专业概念 |

### 7.2 用户特殊要求

如用户在对话中提及以下需求，应在输入包中补充：

- **特定风格要求**：如"模仿某篇文章的风格"
- **特定结构要求**：如"采用问题-解决方案结构"
- **特定案例要求**：如"必须包含某公司的案例"
- **特定数据要求**：如"需要2024年的最新数据"
- **特定受众要求**：如"面向初学者"或"面向专家"

### 7.3 历史上下文

如当前任务与历史任务相关，可补充：

- 同系列历史文章链接
- 用户之前对该主题的反馈
- 之前版本的修改记录

### 7.4 判断标准

主Agent在判断是否补充内容时，应考虑：

1. **是否影响文章质量？** → 是则补充
2. **是否用户明确要求？** → 是则补充
3. **是否W1执行必需？** → 是则补充
4. **补充成本是否过高？** → 过高则记录原因后跳过
