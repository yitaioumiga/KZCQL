# 差异评审Agent (R3) 输入包模板

## 一、Agent 概述

- **代号**: R3
- **角色定位**: 迭代评审阶段的效率优化者，只评审修改涉及的部分
- **核心职责**: 对比修改前后的文章差异，仅针对W2修改涉及的部分进行评审，避免全量重复评审，提升迭代效率

---

## 三、调用前验证清单

- [ ] **自身规范文件已提供** — 文件路径正确且文件存在
- [ ] **修改前文章已提供** — 文件存在且为有效的Markdown格式
- [ ] **修改后文章已提供** — 文件存在且为有效的Markdown格式
- [ ] **评分报告已提供** — 包含明确的维度分数和评审意见
- [ ] **修改说明已提供** — W2输出的修改说明文件存在且内容完整

---

## 四、调用后验证清单

- [ ] **差异评审报告已输出** — 文件格式正确，包含修改部分的逐项评审
- [ ] **更新后的维度分数已输出** — 仅更新受修改影响的维度分数，未修改维度保持原分
- [ ] **修改效果评估已输出** — 明确评估修改是否解决了上一轮指出的问题
- [ ] **是否需要回退R2的判定已输出** — 当修改幅度过大或引入新问题时，明确判定是否需要回退到R2全量评审

---

## 五、错误处理

### 5.1 自身规范文件缺失

- **行为**: **阻断调用**
- **处理**: 主Agent必须先定位自身规范文件路径，确认文件存在后再构造输入包调用R3

### 5.2 修改幅度过大（>30%）

- **行为**: **自动回退到R2全量评审**
- **处理**: 如果R3在对比修改前后文章时发现修改幅度超过全文30%，应立即终止差异评审，通知主Agent回退到R2进行全量评审，确保评审覆盖完整

### 5.3 修改说明缺失

- **行为**: **要求W2补充**
- **处理**: 如果W2未输出修改说明或修改说明不完整，主Agent应要求W2补充修改说明后再调用R3

## 七、建议补充内容（主Agent自由裁量）

> **说明**：以下内容非强制要求，主Agent可根据具体工作场景判断是否需要补充。

### 7.1 场景化补充建议

| 场景 | 建议补充内容 | 补充原因 |
|------|-------------|----------|
| 修改幅度接近阈值 | 阈值判断标准 | 帮助R3决定是否回退R2 |
| 涉及多个维度 | 维度影响分析 | 帮助R3评估修改影响范围 |
| 新增大量内容 | 新增内容来源说明 | 帮助R3验证新增内容质量 |
| 删除大量内容 | 删除原因说明 | 帮助R3判断是否过度删除 |
| 结构调整 | 结构调整意图 | 帮助R3评估结构合理性 |

### 7.2 重点关注提示

如修改涉及以下内容，可提示R3重点关注：

- **事实性修改**：提示R3重新核查事实
- **风格调整**：提示R3关注D4评分变化
- **论点调整**：提示R3关注D3一致性
- **配图修改**：提示R3关注D7评分

### 7.3 历史对比

如有以下材料，可补充供R3参考：

- 上一轮修改的具体位置
- 上一轮修改的效果评估
- 用户上一轮修改的反馈

### 7.4 判断标准

主Agent在判断是否补充内容时，应考虑：

1. **是否帮助R3准确评估修改效果？** → 是则补充
2. **是否影响回退R2的决策？** → 是则补充
3. **是否有助于识别新问题？** → 是则补充
4. **是否会导致R3过度关注某一方面？** → 是则谨慎补充
