# R1 事实核查Agent 输入包模板

## 一、Agent 概述

- **代号**: R1
- **角色定位**: 评审专家组第一道关卡，与原始材料/网络信息交叉比对
- **核心职责**: 逐项检查23项一票否决项，任何一项触发即判定FAIL

## 二、输入包定义

### 2.1 [MUST] 必须输入

| 字段 | 来源路径 | 格式 | 验证方式 |
|------|----------|------|----------|
| **自身规范文件** | `/workspace/KZCQL/02_子Agent规范/评审专家组/事实核查Agent.md` | Markdown | 文件存在，子Agent启动时必须读取 |
| **当前日期** | 主Agent上下文 | YYYY-MM-DD格式 | **必需（V-20数据时效性验证）** |
| 待评文章 | W1/D1输出 | Markdown文本 | 非空，完整文章 |
| 一票否决项清单 | `/workspace/KZCQL/01_共享知识库/后置评审规则/一票否决项.md` | Markdown | 文件存在 |
| 用户原始请求 | 用户输入 | 文本 | 非空（用于V-19/V-04/V-13验证） |
| 配图文件路径列表 | W1/D1输出 | 文件路径数组 | 非空（用于V-16/V-17/V-18检查） |

### 2.2 [SHOULD] 建议输入

| 字段 | 来源路径 | 格式 | 缺失时的默认行为 |
|------|----------|------|------------------|
| 原始材料 | 用户输入 | 文本/链接/文件 | 相关否决项标记为"无法验证" |
| 用户身份档案 | `/workspace/KZCQL/01_共享知识库/作者事实档案.md` | Markdown | V-12标记为"无法验证" |
| 通用配图设计规则 | `/workspace/KZCQL/01_共享知识库/通用配图设计规则.md` | Markdown | V-16/V-17检查精度降低 |

### 2.3 [MAY] 可选输入（条件触发）

| 字段 | 来源路径 | 触发条件 | 缺失时的默认行为 |
|------|----------|----------|------------------|
| 已发布系列文章 | 用户指定 | 系列文章时 | V-02跨篇一致性标记为"无法验证" |
| 作者事实档案 | `/workspace/KZCQL/01_共享知识库/作者事实档案.md` | 涉及作者个人经历时 | V-02标记为"无法验证" |
| 历史核查记录 | 上一轮R1输出 | 多轮核查时 | 无法追踪重复问题 |

## 三、调用前验证清单

- [ ] 自身规范文件路径已包含在Task query中
- [ ] 待评文章为完整Markdown文本
- [ ] 一票否决项清单文件存在
- [ ] 用户原始请求已提供
- [ ] 配图文件路径列表已提供（至少包含文章中引用的所有图片路径）

## 四、调用后验证清单

- [ ] 输出包含PASS或FAIL判定
- [ ] 23项一票否决项全部有检查结果
- [ ] 每个FAIL判定附带具体证据（原文引用+对照来源）
- [ ] 存疑内容已标注
- [ ] 输出格式为Markdown

## 五、错误处理

### 5.1 文件不存在

| 文件 | 处理方式 |
|------|----------|
| 自身规范文件 | **阻断**：终止调用，报告文件缺失 |
| 一票否决项清单 | **阻断**：终止调用，R1无法执行核查 |

### 5.2 输出格式不符

| 问题 | 处理方式 |
|------|----------|
| 缺少PASS/FAIL判定 | 要求R1补充判定 |
| 否决项未逐项检查 | 要求R1补充缺失项的检查 |
| FAIL判定无证据 | 要求R1补充具体证据 |

### 5.3 Agent执行失败

| 场景 | 处理方式 |
|------|----------|
| R1返回FAIL | 进入重写流程（W1重写或W2修改） |
| R1超时 | 重试1次，仍失败则向用户报告 |

## 七、建议补充内容（主Agent自由裁量）

> **说明**：以下内容非强制要求，主Agent可根据具体工作场景判断是否需要补充。

### 7.1 场景化补充建议

| 场景 | 建议补充内容 | 补充原因 |
|------|-------------|----------|
| 引用外部数据 | 原始数据来源链接 | 验证数据准确性 |
| 涉及人物言论 | 言论原始出处 | 验证引用真实性 |
| 涉及历史事件 | 历史背景资料 | 验证时间线准确性 |
| 涉及统计数据 | 统计报告原文 | 验证数字准确性 |
| 涉及技术参数 | 官方技术文档 | 验证参数正确性 |

### 7.2 高风险内容标记

如文章包含以下内容，建议补充额外核查材料：

- **敏感政治内容**：补充权威媒体报道
- **医疗健康建议**：补充医学文献来源
- **金融投资建议**：补充免责声明和数据来源
- **法律相关陈述**：补充法律条文原文

### 7.3 用户关注点

如用户在前期沟通中特别强调某些内容，可补充：

- 用户特别关注的论点/数据
- 用户明确要求核查的内容
- 用户提供的质疑点

### 7.4 判断标准

主Agent在判断是否补充内容时，应考虑：

1. **是否影响事实核查准确性？** → 是则补充
2. **是否存在高风险内容？** → 是则补充
3. **R1是否有能力独立验证？** → 否（需补充材料）则补充
4. **补充成本是否过高？** → 过高则标记"无法验证"
