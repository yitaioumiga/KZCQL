# R2 全量评审Agent 输入包模板

## 一、Agent 概述

- **代号**: R2
- **角色定位**: 评审专家组核心评分者，对文章进行12维度全量打分
- **核心职责**: 严格对标卡兹克风格，输出每个维度的分数、扣分理由和改进建议

## 三、调用前验证清单

- [ ] 自身规范文件路径已包含在Task query中
- [ ] 待评文章已通过R1事实核查（R1=PASS）
- [ ] 评分体系文件存在
- [ ] 参考范文已提供
- [ ] 配图文件路径列表已提供
- [ ] R1事实核查报告已提供
- [ ] 用户原始请求已提供

## 四、调用后验证清单

- [ ] 输出包含12维度评分（D1-D12）
- [ ] 每个维度有分数和具体扣分/加分理由
- [ ] 包含卡兹克风格专项检查结果
- [ ] 评分引用了评分体系中的确切标准
- [ ] 包含改进建议
- [ ] 输出格式为Markdown + JSON摘要

## 五、错误处理

### 5.1 文件不存在

| 文件 | 处理方式 |
|------|----------|
| 自身规范文件 | **阻断**：终止调用，报告文件缺失 |
| 评分体系 | **阻断**：终止调用，R2无法执行评分 |
| R1事实核查报告 | **阻断**：终止调用，D9评分无法执行 |

### 5.2 输出格式不符

| 问题 | 处理方式 |
|------|----------|
| 维度评分缺失 | 要求R2补充缺失维度的评分 |
| 评分未引用标准 | 要求R2补充标准引用 |
| 无改进建议 | 要求R2补充改进建议 |
| 评分体系错误（非100分制） | 要求R2使用正确的评分体系重新评分 |

### 5.3 Agent执行失败

| 场景 | 处理方式 |
|------|----------|
| R2超时 | 重试1次，仍失败则向用户报告 |
| R2评分异常（所有维度分数相同） | 怀疑子Agent未独立判断，重新调用 |

## 七、建议补充内容（主Agent自由裁量）

> **说明**：以下内容非强制要求，主Agent可根据具体工作场景判断是否需要补充。

### 7.1 场景化补充建议

| 场景 | 建议补充内容 | 补充原因 |
|------|-------------|----------|
| 新类型文章 | 同类优秀范文 | 帮助R2建立评分基准 |
| 风格争议 | 多篇参考范文 | 帮助R2理解风格差异 |
| 技术深度争议 | 领域专家评价标准 | 帮助R2评估专业度 |
| 结构创新 | 结构创新案例 | 帮助R2判断是否有效创新 |
| 跨文化内容 | 文化背景说明 | 帮助R2理解文化差异 |

### 7.2 评审重点提示

如文章存在以下特点，可在输入包中提示R2重点关注：

- **用户特别强调的维度**：如"请重点关注可读性"
- **历史问题维度**：如"上次D4评分较低，请重点关注"
- **争议性内容**：提示R2注意平衡性评分
- **创新尝试**：提示R2注意创新性评分

### 7.3 对比基准

如有以下材料，可补充供R2参考：

- 同主题历史文章（对比进步/退步）
- 竞品文章（对比差异化）
- 用户提供的标杆文章（对比差距）

### 7.4 判断标准

主Agent在判断是否补充内容时，应考虑：

1. **是否影响评分准确性？** → 是则补充
2. **是否有助于R2理解上下文？** → 是则补充
3. **是否是用户关注点？** → 是则补充
4. **补充是否会导致评分偏差？** → 是则谨慎补充
