# 迭代修改Agent (W2) 输入包模板

## 一、Agent 概述

- **代号**: W2
- **角色定位**: 写作组精准修改者，针对评分报告中1-2个最高优先级问题进行最小化修改
- **核心职责**: 基于评审反馈，对文章进行精准、最小化的定向修改，避免过度改写导致质量回退

---

### 2.1 [MUST] 必须输入

| 字段 | 来源路径 | 格式 | 验证方式 |

- [ ] **自身规范文件已提供** — 文件路径正确且文件存在
- [ ] **评分报告已提供** — 包含明确的维度分数和修改建议
- [ ] **当前文章已提供** — 文件存在且为有效的Markdown格式

---

## 四、调用后验证清单

- [ ] **修改后的文章已输出** — 文件格式正确，Markdown结构完整
- [ ] **修改说明已输出** — 明确列出本次修改了哪些内容、对应哪个评分问题
- [ ] **配图已更新**（如涉及配图修改） — 新配图文件已生成且路径正确
- [ ] **修改幅度合理** — 仅针对1-2个最高优先级问题进行修改，未出现大范围改写
- [ ] **未引入新问题** — 修改后的文章未破坏原有质量

## 五、错误处理

### 5.1 自身规范文件缺失

- **行为**: **阻断调用**
- **处理**: 主Agent必须先定位自身规范文件路径，确认文件存在后再构造输入包调用W2

### 5.2 评分报告缺失

- **行为**: **阻断调用**
- **处理**: 主Agent必须先完成R2或R3评审，获得评分报告后再调用W2

### 5.3 修改幅度过大

- **行为**: **要求W2缩小范围**
- **处理**: 如果W2输出的修改说明显示修改涉及超过2个问题或大范围重写，主Agent应要求W2重新聚焦于最高优先级的1-2个问题，执行最小化修改

### 5.4 配图修改失败

- **行为**: **重新生成**
- **处理**: 如果W2修改配图后生成的图片不符合通用配图设计规则，主Agent应要求W2重新生成配图

## 七、建议补充内容（主Agent自由裁量）

> **说明**：以下内容非强制要求，主Agent可根据具体工作场景判断是否需要补充。

### 7.1 场景化补充建议

| 场景 | 建议补充内容 | 补充原因 |
|------|-------------|----------|
| 修改范围大 | 修改优先级排序 | 帮助W2聚焦关键问题 |
| 涉及多维度 | 维度间依赖关系说明 | 帮助W2理解修改影响 |
| 用户特殊要求 | 用户具体修改意见 | 确保符合用户预期 |
| 历史修改记录 | 之前修改的教训 | 避免重复犯错 |
| 配图问题 | 配图设计参考 | 帮助W2重新设计配图 |

### 7.2 修改策略提示

如评分报告中有以下情况，可补充提示：

- **多个P0问题**：提示W2优先解决哪个
- **相互矛盾的建议**：提示W2如何取舍
- **模糊的建议**：补充具体化说明
- **超出W2职责的建议**：提示W2忽略或转交

### 7.3 用户反馈

如用户有以下反馈，应补充到输入包：

- 用户对特定段落的不满
- 用户希望保留的内容
- 用户的风格偏好调整

### 7.4 判断标准

主Agent在判断是否补充内容时，应考虑：

1. **是否帮助W2精准定位问题？** → 是则补充
2. **是否避免W2误解评分报告？** → 是则补充
3. **是否符合用户预期？** → 是则补充
4. **是否会限制W2的修改灵活性？** → 是则谨慎补充
