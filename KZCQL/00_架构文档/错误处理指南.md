# 错误处理指南

> 版本: v1.0 | 日期: 2026-05-22
> 关联: 主Agent编排指南.md | 输入包模板使用指南.md

## 一、概述

本指南为主Agent在调用子Agent过程中遇到错误时提供标准化的处理流程。每个输入包模板中都包含简化的错误处理章节，本指南提供完整的错误分类、决策树和参考文档映射。

## 二、错误类型分类

### 2.1 错误分类表

| 错误类型 | 编号 | 严重程度 | 典型场景 |
|----------|------|----------|----------|
| 文件不存在 | ERR-F01 | P0-阻断 | 子Agent规范文件或规则文件路径错误 |
| 输入不完整 | ERR-I01 | P0-阻断 | 必须输入字段缺失 |
| 输出格式不符 | ERR-O01 | P1-重试 | 子Agent输出缺少必要字段或格式错误 |
| 输出内容为空 | ERR-O02 | P1-重试 | 子Agent返回空内容或Skill调用返回空 |
| Agent执行失败 | ERR-A01 | P1-重试 | 子Agent超时或内部错误 |
| 评分体系错误 | ERR-S01 | P1-重试 | 评分使用错误的分制或标准 |
| 验证失败 | ERR-V01 | P2-降级 | 调用后验证不通过 |
| 迭代无提升 | ERR-L01 | P2-终止 | 连续多轮迭代无分数提升 |

### 2.2 严重程度定义

| 级别 | 含义 | 处理方式 |
|------|------|----------|
| P0-阻断 | 必须立即解决，否则无法继续 | 终止当前步骤，修复后重新执行 |
| P1-重试 | 可通过重试解决 | 重试1次，仍失败则向用户报告 |
| P2-降级 | 可降级处理 | 记录问题，降级后继续执行 |

## 三、错误处理决策树

### 3.1 文件不存在 (ERR-F01)

```
文件不存在
├── 是自身规范文件？
│   ├── 是 → 终止调用，检查02_子Agent规范/目录结构
│   └── 否 ↓
├── 是01_共享知识库规则文件？
│   ├── 是 → 终止调用，检查01_共享知识库/目录结构
│   └── 否 ↓
├── 是04_工作区产出文件？
│   ├── 是 → 检查文件命名和归档路径是否正确
│   └── 否 ↓
└── 是可选输入文件？
    ├── 是 → 跳过该文件，记录原因，继续执行
    └── 否 → 终止调用，向用户报告
```

### 3.2 输入不完整 (ERR-I01)

```
输入不完整
├── 缺少[MUST]字段？
│   ├── 是 → 终止调用，补充缺失字段后重新调用
│   └── 否 ↓
├── 缺少[SHOULD]字段？
│   ├── 是 → 记录原因，使用默认值继续执行
│   └── 否 ↓
└── 缺少[MAY]字段？
    ├── 条件已触发？
    │   ├── 是 → 记录原因，降级执行
    │   └── 否 → 正常，该字段不需要
```

### 3.3 输出格式不符 (ERR-O01)

```
输出格式不符
├── 缺少必要字段？
│   ├── 是 → 要求子Agent补充缺失字段
│   └── 否 ↓
├── 格式错误？
│   ├── 是 → 要求子Agent按正确格式重新输出
│   └── 否 ↓
└── 内容与规范不符？
    ├── 是 → 要求子Agent参照规范重新执行
    └── 否 → 检查是否为规范更新导致，如是则更新规范
```

### 3.4 Agent执行失败 (ERR-A01)

```
Agent执行失败
├── 首次失败？
│   ├── 是 → 重试1次
│   │   ├── 重试成功 → 继续执行
│   │   └── 重试失败 → 向用户报告，请求人类介入
│   └── 否 ↓
├── 连续失败（≥2次）？
│   ├── 是 → 向用户报告，请求人类介入
│   └── 否 ↓
└── 是Skill调用失败？
    ├── 是 → 检查Skill是否可用，尝试备用方案
    └── 否 → 检查子Agent规范是否正确
```

### 3.5 评分体系错误 (ERR-S01)

```
评分体系错误
├── 使用了错误的分制？
│   ├── 是 → 要求子Agent使用正确的分制重新评分
│   │   ├── 110分制 → 120分制（十维度）
│   │   ├── 100分制 → 120分制（十维度）
│   │   └── 12维度 → 十维度（D1-D10）
│   └── 否 ↓
├── 维度分值分配不一致？
│   ├── 是 → 统一使用架构专家Agent.md中定义的分值标准
│   │   └── D1=15, D2=15, D3=15, D4=10, D5=15, D6=15, D7=10, D8=10, D9=10, D10=10
│   └── 否 ↓
├── 评分标准引用过时？
│   ├── 是 → 更新为最新评分体系.md中的标准
│   │   └── 检查01_共享知识库/后置评审规则/评分体系.md版本
│   └── 否 ↓
└── 评分与评级不匹配？
    ├── 是 → 检查R4评级判定逻辑，确保与评分体系一致
    └── 否 → 记录异常，继续执行
```

### 3.6 输出内容为空 (ERR-O02)

```
输出内容为空
├── 是首次调用？
│   ├── 是 → 重试1次，检查输入是否完整
│   │   ├── 重试成功 → 继续执行
│   │   └── 重试失败 → 检查输入包是否缺少关键字段
│   └── 否 ↓
├── 输入包中[MUST]字段缺失？
│   ├── 是 → 补充缺失字段后重新调用
│   │   └── 参考ERR-I01处理流程
│   └── 否 ↓
├── 是Skill调用返回空？
│   ├── 是 → 检查Skill可用性，尝试降级方案
│   │   └── 如GenerateImage失败，尝试使用默认配图
│   └── 否 ↓
└── 子Agent执行异常？
    ├── 是 → 向用户报告，请求人类介入
    └── 否 → 记录异常，标记为系统错误
```

### 3.7 迭代无提升 (ERR-L01)

```
迭代无提升
├── 连续无提升计数 ≥ 2？
│   ├── 是 → 终止迭代，向用户报告
│   │   └── 建议：人工终审或重写
│   └── 否 ↓
├── 连续无提升计数 ≥ 3？
│   ├── 是 → 强制终止，建议W1重写
│   └── 否 → 继续迭代
```

## 五、错误记录规范

每次遇到错误并处理后，主Agent应记录以下信息：

```markdown
## 错误处理记录

**时间**: YYYY-MM-DD HH:MM
**Agent**: {Agent代号}
**错误类型**: {ERR-XX-XX}
**严重程度**: {P0/P1/P2}
**错误描述**: {具体描述}
**处理方式**: {采取的处理措施}
**处理结果**: {成功/失败/降级}
**后续建议**: {预防措施}
```
