### 步骤4：生成随笔素材细索引

**目标**：对F1-F4（共约250万字）分批生成摘要索引，采用多层次索引架构

**索引架构设计**（参考RAG分层检索最佳实践）：

```
Level 0（粗索引）：素材索引.md ← 已有，按主题/时间/类型分类
Level 1（细索引）：细索引/F1-F4_摘要索引.md ← 本次新建，按时间段摘要
Level 2（原文）：随笔素材/F1-F4.md ← 原始文件，按需查阅
```

**细索引格式规范**（每个文件统一格式）：

```markdown
# F[n] 摘要索引

> 生成日期：YYYY-MM-DD
> 原始文件：F[n]文件名.md
> 原始大小：约XXX KB
> 摘要粒度：按[时间段/主题段落]分批

## 摘要列表

### [时间段/段落标识]（约L起-L止）
- **时间范围**：YYYY-MM-DD ~ YYYY-MM-DD
- **核心话题**：2-3个关键词
- **内容摘要**：50-100字概括
- **可用素材**：
  - 📖 个人经历：[简述]
  - 💡 思考洞察：[简述]
  - 🔧 方法论：[简述]
  - ❤️ 情感记录：[简述]
- **适合文章场景**：[开头故事/方法论输出/情感共鸣/...]
```

**执行方式**：
- 使用子Agent（general_purpose_task）分别处理F1-F4
- 每个子Agent读取原始文件，按时间段/主题段落生成摘要
- 摘要粒度：每10-20行日志为一个摘要单元
- 输出4个细索引文件

**涉及文件**：
- 创建 `随笔素材/细索引/F1_摘要索引.md`
- 创建 `随笔素材/细索引/F2_摘要索引.md`
- 创建 `随笔素材/细索引/F3_摘要索引.md`
- 创建 `随笔素材/细索引/F4_摘要索引.md`

## 四、假设与决策

1. **索引粒度决策**：细索引按10-20行日志为一个摘要单元（而非每条日志），平衡精度和可维护性
2. **内容工程定义**：内容工程 = 写作 + Skill + 开源项目 + 个人网站 + 内容产品（不含科研项目）
3. **选题A确认**：用户已接受选题方向A（内容工程全貌），先归档待后续执行
4. **子Agent并行**：F1-F4细索引生成使用4个并行子Agent，提高效率
5. **MTC配置优化范围**：仅更新KZCQL系统内的Agent配置和编排器，不涉及外部MTCQL仓库
