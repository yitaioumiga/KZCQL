## 一、前置撰写规则.md 文件体量评估

### 1.1 基础数据

| 指标 | 值 |
|------|-----|
| 文件大小 | **74,895 字节（约 73KB）** |
| 总行数 | **1,654 行** |
| 章节数 | §1-§18 + S19/S20/S21（共 21 个顶级章节） |
| 子章节数 | 约 80+ 个二级/三级标题 |

### 1.2 单次读取限制分析

**结论：已超出单次读取限制，强烈建议拆分。**

- Read 工具单次最多读取约 2000 行，但实际推荐在 200-250 行范围内以保证上下文质量
- 当前 1654 行虽然技术上在单次读取上限内，但 73KB 的内容量意味着 Agent 在单次读取后上下文占用极大
- W1 Agent 需要完整读取此文件，同时还要读取作者事实档案、系列文章写作规范等，总上下文负担过重
- **风险**：Agent 在处理如此大的规则文件时，容易遗漏后半部分的规则（特别是 S19/S20/S21 位于文件末尾第 1491-1654 行）

### 1.3 拆分建议 [严重程度：高]

建议将前置撰写规则.md 拆分为以下文件：

| 文件 | 内容 | 预估行数 |
|------|------|----------|
| 前置撰写规则.md（核心） | §1-§7（标题、语言、结构、配图、架构图、图片、技术规范） | ~870 行 |
| 前置撰写规则_经验与素材.md | §8-§10（写作前必问、用户身份档案、经验教训） | ~170 行 |
| 前置撰写规则_内容增强.md | §11-§14（方法论、结构增强、内容深度、理财规范） | ~190 行 |
| 前置撰写规则_多平台适配.md | §15-§18 + S19/S20/S21（APAG、故事、CTA、跨领域、开头故事、结尾强化、人工校验） | ~260 行 |
