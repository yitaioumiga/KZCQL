## 一、Git版本控制问题与解决方案

### 1.1 当前问题

| 问题 | 现状 | 风险 |
|------|------|------|
| 环境重置 | 我的运行环境会定时重置 | `/workspace`虽然持久化，但git历史可能丢失 |
| 本地单点 | 只有本地git仓库 | 磁盘损坏、误操作都会导致数据丢失 |
| 历史丢失 | 之前的git提交已消失 | 无法回溯到之前的版本 |

### 1.2 解决方案：远程仓库 + 本地备份

#### 方案A：GitHub私有仓库（推荐）

```bash
# 1. 在GitHub创建私有仓库（如：kazik/KZCQL）
# 2. 本地关联远程仓库
git remote add origin https://github.com/kazik/KZCQL.git

# 3. 推送所有分支
git push -u origin main

# 4. 定期推送（每次重要修改后）
git push
```

**优点**：
- 免费私有仓库
- 版本历史云端保存
- 支持多设备同步
- 可视化commit历史

**缺点**：
- 需要GitHub账号
- 代码托管在第三方

#### 方案B：Gitee码云（国内替代）

```bash
git remote add origin https://gitee.com/kazik/KZCQL.git
git push -u origin main
```

**优点**：国内访问快，同样免费私有仓库

#### 方案C：本地双备份

```bash
# 在另一个目录创建裸仓库备份
git clone --bare /workspace/KZCQL /backup/KZCQL-backup.git

# 定期同步
cd /workspace/KZCQL
git push /backup/KZCQL-backup.git main
```

**建议**：GitHub（主）+ 本地备份（辅）

## 二、架构升级总体规划

### 2.1 升级原则

| 原则 | 说明 |
|------|------|
| **可解离** | 新功能作为可选模块，可随时关闭 |
| **渐进式** | 分阶段实施，每阶段可独立验证 |
| **可回退** | 每个阶段都有明确的回退方案 |
| **文档先行** | 先写规范，再写代码 |

### 2.2 升级范围

**新增Agent（2个）**：
- D2 调研智能体（Research Agent）
- D3 角度挖掘智能体（Angle Agent）

**现有Agent保持不变**：
- W1, W2, R1, R2, R3, R4, I1, E1, A1, D1（全部不动）

### 2.3 升级后流程（可选模式）

```
模式A：精简模式（现有流程，默认）
用户主题 → W1 → R1 → R2 → R4 → 交付

模式B：完整模式（新增D2+D3）
用户主题 → D2调研 → D3角度选择 → W1 → R1 → R2 → R4 → 交付
                              ↑
                         人类选择角度
```

## 附录：Git远程仓库设置指南

### 创建GitHub私有仓库

1. 登录 https://github.com
2. 点击右上角 "+" → "New repository"
3. 填写信息：
   - Repository name: `KZCQL`
   - Description: `KZCQL多智能体内容生产系统`
   - Visibility: **Private**（重要！）
   - 勾选 "Add a README file"
4. 点击 "Create repository"

### 本地关联远程仓库

```bash
cd /workspace/KZCQL

# 添加远程仓库（替换为您的用户名）
git remote add origin https://github.com/您的用户名/KZCQL.git

# 推送代码
git push -u origin main

# 验证
git remote -v
```

### 定期推送脚本

```bash
#!/bin/bash
# /workspace/backup.sh

cd /workspace/KZCQL
git add .
git commit -m "backup: $(date '+%Y-%m-%d %H:%M:%S')"
git push origin main

echo "Backup completed at $(date)"
```
